Creating a Badge Type

The scope of the badging function is defined through:
  • Badge Type -Identifies the various badges available in the user's org.
  • Badge Workflow-Defines the assignment rules.
  • Badges-Enable specific Accounts and Contacts opportunities for custom pricing.
  • Access Permissions-Restrict visibility across various records.

    Staff users can define a numerical score for Badges that accumulate on the Account or Contact record, thus encouraging and tracking member engagement and enabling tiered pricing.

  1. Navigate to the Badge Types tab in the eStore app and click New.
  2. Enter the Badge Name and click Save.
    Best practices recommend populating the Short Description and Image Location fields. The Short Description is used on reports and provides context to end users looking at a list of Contacts assigned to various badges.
  3. Enter the Public Description.
    The content in the Public Description field is displayed in the Badges tab on the Profile page of the Community Portal.
    Optionally, assign a numerical value to the Badge via the Engagement Score.
  4. Check the Is Active and Is Published checkboxes in order for the Badge Type to be available for assignment and displayed on the user's Profile page in the Community Portal. Click Save.
The newly created Badge Type is available for assignment to an Account or Contact.
