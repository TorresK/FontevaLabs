Creating a Price Rule

Price Rules enable System Admins and staff users to manage dynamic pricing primarily through assigning Badge Types and applying discounts. The default Price Rule is created when the item is created; creating a new Price Rule populates the Item record with a new custom Price Rule. The newly created Price Rule is applied to all users who meet the acceptance criteria of the Price Rule variables.
Note: All new items inherit the Price Rule configurations that are linked to that specific Item Class. After a new item is created, item pricing rules can be refreshed to include the new default Price Rule assigned to the Item Class.
  • Create an Item
    Note: The Not for Individual Sale flag is available only on the item. Otherwise, ticket types, schedule items, and sponsor packages all have an associated, mirror-image item created upon saving the respective record. Occasionally, there are instances when it is helpful to compare the two records to check for any missing configurations.
  1. Navigate to the Price Rules related list on the Item record and click the New Price Rule button.
  2. Enter a Price Rule Name and Price.
    Default is set to the Item Name and Item Price entered on the Item record. When using a Source Code, make sure the Price entered is the Price customers will pay after the Discount Code is applied.
  3. If required, select the applicable Badge Types, Subscription Plans, or Source Codes.
  4. Optionally, specify the acceptance criteria by populating the Start Date, End Date, Min. Quantity, Max. Quantity, and Max. Num. Available fields.
    If the Max. Quantity field is left null, there will be no constraint on the maximum quantity of items purchased. If the Min. Quantity field is left null, there will be no constraint on the minimum quantity of items allowed for purchase. Populating the Max. Num. Available field defines the maximum number of times the Price Rule can be applied; generally used to incentivize customers to purchase products early or for flash/date-driven sales.
  5. For item price rule support, enter the Shopping Cart Item that must be present in order for customers to receive beneficial pricing on the item.
  6. Populate the Object, Field, Operator, and Value fields to establish Price Rule Variables.
    The Object field identifies the object hosting the field attribute to assign the Price Rule. The Field value identifies the attribute to assign to the Price Rule. The Operator field identifies the operation that helps define the criteria used to evaluate the attribute's value for the Price Rule. The Value field displays the attribute's value for the Price Rule.
    Leave the Value as null to exclude Price Rule Variables.
  7. Check the Is Active checkbox and click Save.
    Only active Price Rules are applied to new purchases related to the item.
A new Price Rule is created and will appear in the Pricing Rules related list on the related Item record.
