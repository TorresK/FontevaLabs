Default Shipping Rates

The eStore app has a Default Shipping Rate feature that ensures customers can successfully complete a purchase even if there is no Shipping Rate defined for their region or item weight/value. The only time a customer is prevented from making a purchase with shipping is if they have not entered a shipping address at all, or if their shipping country is in a Will Not Ship region.
Note: A Default Shipping Rate can be defined for each Shipping Carrier, with a maximum of one default per carrier. When the Is Default checkbox is selected for the Shipping Rate, staff users cannot associate a region with the rate. Additionally, the Minimum Weight and Maximum Weight fields can be left blank.
Customers will see the Default Rate in the eStore checkout if:
  • The shipping item has the Require Shipping checkbox enabled and the Shipping Rate Basis is set to Item Weight, but the Item Weight is missing or given a value of "0."
  • The total weight or value of the item (item x quantity) does not match to an existing Shipping Rate.
  • The shipping address country is misspelled, or does not match to an existing Shipping Rate, and there are no rates set up for the Default Shipping Region.
