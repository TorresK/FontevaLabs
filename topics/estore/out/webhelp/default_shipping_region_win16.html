Creating a Default Shipping Region

The Default Shipping Region matches any shipping destination that is not explicitly defined in another region; in other words, a region that covers "everywhere else."
Note: The Default Shipping Region is different from the Default Shipping Rate. The System Admin can define multiple shipping rates on the Default Shipping Region, whereas the Default Shipping Rate is defined on the Shipping Carrier and applies the same rate across all item weights/values and shipping addresses.
  • Create the Shipping Carrier and Delivery Method
  1. Navigate to the Shipping Regions tab in the eStore app and click New.
  2. Enter the Shipping Region Name.
    The Shipping Region Name appears in the Shipping Rates lookup. Optionally, enter a Display Name and Description.
  3. Check the Is Default Region checkbox and click Save.
    Leave the Address Field Values (CSV) field blank when enabling Is Default Region. The checkbox is unchecked by default.
With the Is Default Region feature enabled, staff users can create shipping rates on a Default Shipping Region. The Default Shipping Region references the Shipping Region field and is then defined as any value that is not found within another Shipping Region.
