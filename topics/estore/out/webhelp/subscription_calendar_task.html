Creating a Calendar Subscription Plan

Calendar Subscription Plans have a defined start and end date. Calendar Subscription Plans can use proration rules to reduce the purchase price for mid-term purchases.

  1. Navigate to Subscription Plans in the eStore app and click New.
  2. Enter a Name for the Subscription Plan, set the Type to Calendar, and check the Is Active checkbox.
  3. Select the Auto Renew Option:
    • Disabled- Turns off auto-renew for all related Subscriptions.
    • Enabled- Allows related Subscriptions to optionally enable auto-renew.
    • Required- Enables auto-renew without option. If selected, then check the Require Payment Method checkbox.
    If the Auto Renew Option is set to Enabled or Required, enter the number of days prior to the subscription's expiration date that customers are invoiced into the Invoice Days Variable field. The Invoice Days Variable field and the Days to Lapse field must have the same value; value must be greater than 0.
  4. Select the Calendar End Day and the Calendar End Month.
    If the Calendar End Day is not specified, the last day of the month is used by default. If a day is selected that does not exist for the selected End Month (Example: '31' for 'February'), then the End Day that appears within the Subscription record will fall in the next month (Example: The End Date would be 'March 3' since the system is counting 31 days after February 1st). This calculation is to ensure a valid date always appears within the subscription record.
    Optionally, select Enable Proration and configure the Proration Rule field: Daily, Weekly, Monthly, Quarterly, or Bi-Annually. The Proration Rule allows for the distribution of the purchase price over a set time period.
  5. Add a Description and click Save.
A Calendar Subscription Plan is created.
