Creating a Store

Stores are fundamental for event registration, renewing subscriptions, and paying invoices in the Community Portal. Staff users can additionally configure payment information, catalogs and catalog items, and guest checkout settings. The system permits one Store for every Site.
Note: Configuring the guest checkout settings match unauthenticated portal users to existing contacts.
  • Create a Business Group
  • Connect to a Payment Gateway
  • Ensure that the user creating the Store has a Role assigned to their User record.
  1. Navigate to the Stores tab in the eStore app and click New.
  2. Enter the Store Name and select the Business Group and Payment Gateway using the lookups.
  3. Check the Enable Guest Checkout checkbox.
    Enabling the guest checkout allows an unauthenticated portal user to process transactions without creating a new user account. Default is unchecked.
  4. Configure the Default Checkout:
    • Guest Checkout - Customers will view the Checkout as Guest page as the first option during checkout. Guest Checkout is the default value when the Enable Guest Checkout field is checked.
    Note: The Create an account checkbox and Create Password text field will display only if Self-Registration is enabled for the community.
    • Account Login - Customers are asked to log in to their account as the first option during checkout. Account Login is the default value if the Enable Guest Checkout field is not checked.
  5. Enter the Payment Confirmation Message to display to users after successfully processing payment on the Checkout page. Click Save.
  • Create an Item Class and the included Items that will be added to the catalog. Then, create a catalog and add it to the Store.
  • Create a Site and enable the eStore on the Site record.
    Note: Every Store must have a catalog in order for public users to make purchases in the Community Portal. Staff users should create the catalog's Item Class along with its items before creating the catalog.
