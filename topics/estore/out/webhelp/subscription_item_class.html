Creating an Item Class for Subscriptions

Item Classes group and set options for items at the class level. All items must have an Item Class.

  1. Navigate to the Item Classes tab in the eStore application and click the New button.
  2. Enter the Name of the Item Class.
    Identify the Business Group via the lookup and enter the Description for the Item Class.
  3. Check the Is Subscription checkbox and click Save.
  4. Check Enable Assignments only if the type of subscription being created involves multiple Contacts under an Account. If Enable Assignments is checked, navigate to the Account record and ensure that Restrict Assignment Visibility is disabled to allow staff users to assign additional Contacts within the Account to a subscription plan, and configure Assignment Roles in the Community Portal.
    If Restrict Assignment Visibility is enabled, then only the logged in user can assign themselves to a subscription plan. If the logged in user has the Is Active and Is Primary checkboxes enabled on the Assignment Role record, then the user can manage their Assignment Role in the Community Portal.
    Click Save.

Set Assignment Roles

If Enable Assignments is checked, then Assignment Roles must be set for the Subscription Plan. Assignment Roles control access to important functions related viewing and modifying subscriptions in the Community Portal.

  1. Navigate to the Assignment Roles related list and click the New Assignment Role button.
  2. Enter the Assignment Role Name to identify the assignees of the subscription and, optionally, enter a Description.
  3. Configure the Assignment Role:
    • Is Active- The Is Active checkbox must be enabled in order to select the assignment role for the assignment.
    • Is Default- When an Assignment Role is not selected, the customer is auto-assigned a Default role that will allow the assignee to view their subscription in the Community Portal.
    • Is Primary- When a customer is assigned to a subscription with Is Primary checked, the customer can modify their subscription in the Community Portal.
    At a minimum, a subscription based Item Class must have at least one Assignment Role with the Is Default and Is Active checkboxes enabled. If the Assignment is not the same as the Contact or Account on the Sales Order, users must navigate to the Sales Order record to identify the Contact or Account purchasing the subscription. All assignees can view the subscriptions that are assigned to them. In order to allow the customer full access to subscription management in the Community Portal, enable Is Primary and Is Active.
  4. Select the Is Active checkbox on the Item Class record and click Save.
    For a subscription based Item Class, an Assignment Role must be created for the Item Class if assignments are enabled in order for the Item Class record to be considered Active.
The Subscription based Item Class is created and Assignment Roles are set.
